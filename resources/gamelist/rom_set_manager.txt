Rom Set Manager = moteur de masquage, regroupement et résolution d’affichage

Il faut penser le menu comme une cascade d’options, avec peu de choix au premier niveau, puis des sous-menus spécialisés.

APIEXPOSE
└── ROM SET MANAGER
    ├── GENERAL
    ├── GROUPING
    ├── REGION & LANGUAGE
    ├── VERSION PRIORITY
    ├── SPECIAL VERSIONS
    ├── ARCADE
    ├── OUTPUT
    └── DEBUG
Structure recommandée des sous-menus
GENERAL
- ROM SET MANAGER ENABLED        switch
- MODE                           select: off / display_only / hide_variants / grouped_view
- APPLY TO CURRENT SYSTEM ONLY   switch
- KEEP ALTERNATIVES ACCESSIBLE   switch

Ici, MODE est le plus important :

off
→ aucun traitement

display_only
→ analyse et rapport, mais aucun masquage

hide_variants
→ masque les variantes non retenues

grouped_view
→ affiche une entrée principale + alternatives accessibles

Ensuite :

GROUPING
- GROUPING METHOD                select: filename / gamelist / dat / hybrid
- NORMALIZE TITLES               switch
- IGNORE ARTICLES                switch
- IGNORE REGION TAGS             switch
- USE CLONE LIST                 switch

Ce sous-menu sert à définir comment APIExpose reconnaît que deux ROMs sont le même jeu.

REGION & LANGUAGE
- REGION PROFILE                 select: france / europe / usa / japan / world / custom
- LANGUAGE PROFILE               select: french / english / multilingual / custom
- REGION BEFORE LANGUAGE         switch
- ACCEPT MULTI REGION            switch
- UNKNOWN REGION POLICY          select: keep / penalize / hide
- UNKNOWN LANGUAGE POLICY        select: keep / penalize / hide

Ici, la cascade importante est :

REGION PROFILE = france
→ France > Europe > World > USA > Japan

LANGUAGE PROFILE = french
→ fr > multi > en > unknown > others

Puis :

VERSION PRIORITY
- PREFER LATEST REVISION         switch
- PREFER RETAIL VERSION          switch
- PREFER RETROACHIEVEMENTS       switch
- PREFER WORLD VERSION           select: first / after_preferred / last
- TIE BREAKER                    select: newest / shortest_name / alphabetical / smallest_file

Ce sous-menu sert quand plusieurs ROMs sont équivalentes.

SPECIAL VERSIONS
- TRANSLATIONS                   select: hide / keep / prefer_if_language_match
- HACKS                          select: hide / keep / prefer_never
- PROTOTYPES                     select: hide / keep
- BETAS                          select: hide / keep
- DEMOS                          select: hide / keep
- HOMEBREW                       select: hide / keep
- AFTERMARKET                    select: hide / keep
- PIRATE / UNLICENSED            select: hide / keep

Je préfère ici des select plutôt que des simples switch, car translation n’est pas juste oui/non.

Exemple :

TRANSLATIONS = prefer_if_language_match

veut dire :

Si la ROM officielle est japonaise
et qu’une traduction française existe
alors la traduction FR peut devenir la ROM principale.

Pour l’arcade :

ARCADE
- ARCADE HANDLING                select: disabled / parent_only / best_clone / parent_clone_group
- PREFER PARENT                  switch
- CLONE REGION PRIORITY          select: world / usa / europe / japan / custom
- KEEP BOOTLEGS                  select: hide / keep
- KEEP HACKS                     select: hide / keep
- KEEP MECHANICAL                select: hide / keep
- KEEP BIOS / DEVICE             select: hide / keep

L’arcade doit être séparé, car MAME/FBNeo n’obéissent pas à la même logique que les consoles.

OUTPUT
- OUTPUT MODE                    select: gamelist_hidden / collection / api_filter / report_only
- WRITE GAMELIST HIDDEN FLAGS    switch
- CREATE CLEAN COLLECTION        switch
- CREATE ALTERNATIVES COLLECTION switch
- BACKUP BEFORE WRITE            switch

Comme tu ne supprimes jamais les ROMs, les sorties possibles sont :

gamelist_hidden
→ ajoute hidden=true sur les variantes

collection
→ crée une collection propre 1 jeu visible par rom groupé

api_filter
→ APIExpose filtre l’affichage sans toucher aux fichiers

report_only
→ génère seulement un rapport

Et enfin :

DEBUG
- GENERATE REPORT                switch
- REPORT FORMAT                  select: json / txt / html
- LOG DECISION SCORE             switch
- LOG GROUP DETAILS              switch
- DRY RUN                        switch

Cascade logique
1. Lire global.apiexpose.romset.enabled.
2. Si OFF, ne rien faire.
3. Lire global.apiexpose.romset.mode.
4. Si display_only, scanner + rapport seulement.
5. Lire grouping_method.
6. Si système arcade, lire arcade_handling.
7. Si arcade_handling != disabled, utiliser parent/clone.
8. Sinon utiliser grouping_method console.
9. Lire region_profile.
10. Lire language_profile.
11. Lire translations.
12. Lire special_versions.
13. Calculer score.
14. Choisir ROM principale.
15. Masquer les variantes selon output_mode.
16. Générer rapport si debug_report=1.

Pour les menus, je garderais cette règle :

Premier niveau = peu d’options compréhensibles.
Sous-menus = options spécialisées.
Pas de suppression, uniquement masquage ou filtrage.
