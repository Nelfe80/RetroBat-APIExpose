# MEM Nomenclature Specification (v2)

## Overview

This document defines the structure, naming and normalization rules for
`.MEM` files: per-game memory event definitions consumed by the RetroBat
ingame runtime.

**The code is the authority.** This specification documents the format as it
is actually parsed by:

1. `wrapper.cpp` (`LoadMemFile` + `ProcessWatchValue`) — the reference
   runtime parser, used by every wrapped RetroArch core;
2. `MameLuaIngameProvider.cs` (APIExpose) — the C# parser used for MAME
   standalone through the `apiexpose_ingame` Lua bridge.

Files are generated by `mem-curator` (APIExpose `tools/mem-curator/`), which
carries the action nomenclature (V11) and the family taxonomy described here.

Version 2 replaces the original draft spec, which described a format
(`memory.variables` section, `equal`/`any` conditions, snake_case rom names)
that was never produced by the generator nor consumed by the runtime.

---

## 1. File Format

Each `.MEM` file is a Lua table returned by the file. Files live in
`resources/ram/<system>/<slug>.MEM`, where `<system>` is the RetroBat folder
name (arcade-like systems are merged under `arcade`).

```lua
-- mem-curator v5.0.0 (2026-07-12) profile=wrapper
return {
  game = {
    title = "Sonic the Hedgehog",
    system = "megadrive",
    system_name = "Genesis/Mega Drive",
    genre = "Platform, Action"
  },

  rom = {
    name = "sonic-the-hedgehog",
    hashes = {
      { hash = "1bc674be034e43c96b86487ac69d9293", label = "Sonic The Hedgehog (USA, Europe).md" },
    }
  },

  events = {
    flow = {
      lifecycle = {
        { address=0XF601, type="u8", condition="eq", value=0X0C, action="GAME_PLAYING", desc="Normal level" },
      },
    },
    scoring = {
      points = {
        { address=0XFE26, type="u32be", condition="change", action="SCORE_STATE", desc="Score" },
      },
    },
  },
}
```

Top-level sections: `game`, `rom`, `events`. There is no `memory` section.

### 1.1 `game`

- `title` (string, required), `system` (RetroBat folder name, required),
  `system_name` (string), `genre` (comma-separated resolved categories).
- Note: the wrapper sniffs the whole file content for the words `megadrive`
  or `genesis` to flag byte-swapped RAM; `game.system` provides that signal.

### 1.2 `rom`

- `name`: canonical slug, lowercase kebab-case (e.g. `sonic-the-hedgehog`).
  It matches the `.MEM` filename and the values of `alias.json`.
- `hashes`: list of `{ hash, label }` from the source database. Informative;
  runtime ROM matching goes through `alias.json`.

### 1.3 Layout constraints (parser contract)

The wrapper parser is text-based, not a Lua VM:

- Entries are split on occurrences of the word `address`. The word
  `address` must therefore never appear outside an entry key (notably not in
  `desc`).
- Category and sub-family are recovered from **indentation**: keep the
  canonical layout `events` (2 spaces) > category (4 spaces) > sub-family
  (6 spaces) > entries (8 spaces).
- Keys are detected as `key=`; a literal `=` inside `desc` is forbidden
  (the generator replaces it with `:`).
- `desc` must be the last field of an entry.

## 2. Event Entry Fields

Canonical field order:

```lua
{ address=0XF601, type="u8", condition="eq", value=0X0C, mask=0X80, action="GAME_PLAYING", color="orange", min=300, max=390, no_log=true, no_survey=true, desc="Normal level" }
```

| Field | Required | Parsed by | Meaning |
| --- | --- | --- | --- |
| `address` | yes | both | Memory address (hex `0X...`). Console systems: address modulo system RAM size. Arcade: flattened SYSTEM_RAM memory-map address. |
| `type` | yes | both | `u8`, `u16be`, `u16le`, `u24be`, `u24le`, `u32be`, `u32le`. Anything else is read as `u8` (1 byte). |
| `condition` | yes | both | See section 3. |
| `value` | for eq/neq | both | Comparison value (defaults to 0 when absent). |
| `mask` | for bit_* | wrapper | Bit mask. Also ANDed onto the read value for non-bit conditions when present. |
| `action` | yes | both | UPPERCASE `[A-Z0-9_]+` action from the V11 nomenclature (section 5). |
| `color` | no | C# | Effect color hint for arcade score-delta events (`green`, `blue`, `orange`, ...). |
| `min` / `max` | no | both | Bounds on the absolute delta (score thresholds). |
| `factor` | no | wrapper | Multiplies the delta before min/max check. |
| `map` | no | wrapper | Inline `{[value]="label"}` display map appended to the log line. |
| `action_map` | no | wrapper | Inline `{[value]="ACTION"}` override; a value absent from the map suppresses the trigger. |
| `is_score` | no | wrapper | Forces score semantics; implicit for the `scoring` category. |
| `no_log` | no | both | `true`: watcher skipped at load time by the wrapper; never published by the C# provider. Omitted when false. Kept in files so end users can re-enable entries by flipping the flag. |
| `no_survey` | no | both | Same effect as `no_log` in the wrapper; reserved for future on-demand reads. Omitted when false. |
| `score_kind` / `score_mask` / `score_encoding` | no | APIExpose | Score semantics parsed from the `.MEM` by the live score aggregator (`/ws/score`). |
| `timer_kind` / `timer_role` / `timer_direction` / `timer_unit` | no | APIExpose | Timer semantics parsed from the `.MEM` by the live timer aggregator (`/ws/timer`). |
| `player` | no | reserved | Player index (LedManager routing, future use). |
| `desc` | yes | both | Short English description. No `address` word, no `=`, no double quotes, max 96 chars, last field. |

The generator's default profile (`wrapper`) does not emit entries whose
`no_log` or `no_survey` is true: both runtimes ignore them entirely.

## 3. Conditions

Recognized by `wrapper.cpp` (`ParseCondType`); anything else never triggers:

| Condition | Trigger |
| --- | --- |
| `change` | Value changed. If `value=` is present and non-zero, only when the new value equals it. Delta bounded by `min`/`max` when present. |
| `eq` (alias `equal`) | Value becomes equal to `value`. |
| `neq` (alias `not_equal`) | Value becomes different from `value`. |
| `increase` | Value went up (delta within `min`/`max` when present). |
| `decrease` | Value went down (delta within `min`/`max` when present). |
| `bit_true` | All bits of `mask` transition to set. |
| `bit_false` | All bits of `mask` transition to clear. |
| `color` | Reserved (value change). |

`any` is NOT a valid condition; the generator normalizes it to `change`.

C# provider nuances: it additionally accepts `ne`, `gt`, `lt`; entries with
`min`/`max` are evaluated as positive BCD-like deltas.

---

## 4. Families

`events` is organized as `category.subfamily`. Known families (taxonomy used
by the generator and by LedManager effect routing):

- `flow.lifecycle`, `flow.settings`, `flow.events`
- `progression.level`, `progression.zone`, `progression.stage`
- `resources.lives`, `resources.health`, `resources.secondary`, `resources.environmental`
- `inventory.items`, `inventory.weapon`
- `scoring.points`, `scoring.collectibles`, `scoring.experience`
- `combat.enemies`, `combat.boss`, `combat.tactical`
- `racing.vehicle`
- `state.temporary`, `state.player`, `state.mount`
- `world_interaction.objects`
- `system.movement`, `system.timer`, `system.memory`, `system.unmapped`, `system.internal`

The `scoring` category implies score semantics in the wrapper (BCD decoding,
no anti-spam auto-mute).

## 5. Action Nomenclature (V11)

Actions are UPPERCASE `[A-Z0-9_]+` strings. The generator maps each action to
its family. Core vocabulary by family:

- **flow.lifecycle** : `TITLE_SCREEN`, `GAME_PLAYING`, `GAME_OVER`,
  `PAUSE_ON`, `PAUSE_OFF`, `DEMO_MODE`, `CONTINUE_SCREEN`,
  `CORPORATE_SCREEN`, `CREDITS_SCREEN`, `INTRO_SCREEN`, `SELECT_SCREEN`,
  `WORLD_MAP`, `LOADING_SCREEN`
- **flow.settings** : `SETTINGS_CHANGED`, `MODE_STATE`
- **flow.events** : `EVENT_TRIGGER`, `BALL_LOCK`, `MULTIBALL_START`, `COMBO_HIT`
- **progression** : `NEW_LEVEL`, `LEVEL_CLEAR`, `PROGRESSION_ZONE`,
  `PROGRESSION_STAGE`, `STAGE_SELECT`, `LAP_STATE`, `RANK_STATE`
- **resources.lives** : `LIVES_STATE`, `LOSE_LIFE`, `GAIN_LIFE`,
  `UNIT_COUNT`, `UNIT_GAIN`, `UNIT_LOSE`
- **resources.health** : `HEALTH_STATE`, `HIT`, `HEAL`, `LOW_HEALTH_WARN`
- **resources.environmental** : `DROWNING`, `ENVIRONMENT_*`
- **resources.secondary** : `RESOURCE_STATE`, `RESOURCE_GAIN`, `RESOURCE_LOSE`
- **inventory.items** : `INVENTORY_ITEM`, `DYNAMIC_INVENTORY`, `TREASURE`,
  `KEY_GET`, `TREASURE_*`
- **inventory.weapon** : `WEAPON_UPGRADE`, `WEAPON_STATE`
- **scoring.points** : `SCORE_STATE`
- **scoring.collectibles** : `COIN_GAIN`, `COIN_LOSE`, `MONEY_STATE`,
  `AMMO_STATE`, `AMMO_GAIN`, `AMMO_LOSE`
- **scoring.experience** : `EXPERIENCE_STATE`
- **combat.enemies** : `BOSS_HIT`, `BOSS_DEFEATED`, `ENEMY_HIT`, `BOMB_FIRED`
- **combat.tactical** : `BATTLE_START`, `BATTLE_END`, `CRITICAL_HIT`,
  `PARRY_SUCCESS`, `KO`, `FATALITY`
- **racing.vehicle** : `CRASH`, `TURBO_BOOST`
- **state.temporary** : `INVINCIBILITY_START/STOP/TIMER`,
  `SPEED_START/STOP/TIMER`, `SHIELD_GAIN/LOST/TIMER`,
  `STATUS_EFFECT_START/STOP`, `TRANSFORMATION`, `TRANSFORMATION_*`,
  `SPECIAL_ACTION`, `PLAYER_STATE_*`
- **state.player** : `JUMPING`, `RUNNING`, `CROUCHING`, `FALLING`,
  `SPINNING`, `SWIMMING`, `ATTACKING`
- **state.mount** : `MOUNT_START`, `MOUNT_STOP`, `MOUNT_STATE`
- **world_interaction.objects** : `OBJECT_INTERACTION`,
  `OBJECT_INTERACTION_CHECKPOINT`, `OBJECT_DESTROYED`, `DOOR_OPENED`,
  `ROOM_DISCOVERED`, `ENVIRONMENT_FORCE`, `OBJECT_INTERACTION_*`
- **system.timer** : `GENERAL_TIMER`, `LEVEL_TIMER`, `BOMB_TIMER`,
  `TIMER_LOW_WARN`, `COMBO_TIMER`
- **system.movement** : `SPEED_STATE`
- **progression.level (dynamic)** : `LEVEL_*` composed tokens (e.g.
  `LEVEL_GREEN_HILL_ZONE_FIRST_ZONE`)

Composed prefixes (`LEVEL_`, `TREASURE_`, `TRANSFORMATION_`,
`PLAYER_STATE_`, `OBJECT_INTERACTION_`, `ENVIRONMENT_`) allow game-specific
suffixes built from labels.

## 6. Runtime Behavior (wrapper.cpp)

- **Warm-up**: the first 120 frames only capture baseline values.
- **Read deduplication**: watchers on the same address/size/endianness share
  one read per frame; unchanged raw values are skipped.
- **Auto-mute**: a non-score watcher that triggers 8 consecutive times with
  gaps of 20 frames or less is permanently muted (infinite-loop signature).
- **Score handling**: `scoring` entries are BCD-decoded; deltas are checked
  against `min`/`max` when present; score events bypass auto-mute.
- **Output**: events are written to the `RetroBatArcadePipe` named pipe as
  `[hh:mm:ss.mmm] [ADDR:0x...] [VAL:0x..] [UDP_OUT] CHANNEL:ACTION |
  SOURCE:desc | VALUE:n | RATE:n | FAMILY:category.subfamily[ | COLOR:color]`.
  Channel is `STATE` for `NEW_LEVEL`/`GAME_STATE`/`*SCREEN*` actions, `SCORE`
  for score entries, `ACTION` otherwise. `FAMILY` carries the V11 family for
  effect routing (LedManager); `COLOR` is only present when the entry defines
  one (wrapper >= 0.310; older parsers ignore everything after `RATE`).
  APIExpose republishes them on `/ws/ingame` with `family` and `color` fields.

## 7. ROM Aliasing (`alias.json`)

Each system folder contains an `alias.json` mapping ROM identities to the
canonical `.MEM` slug:

```json
{
  "Sonic The Hedgehog (USA, Europe)": "sonic-the-hedgehog",
  "sonic the hedgehog (usa, europe)": "sonic-the-hedgehog",
  "1bc674be034e43c96b86487ac69d9293": "sonic-the-hedgehog",
  "1943j": "1943-the-battle-of-midway"
}
```

Rules (wrapper.cpp `searchAlias`):

- one entry per line (the wrapper matches line by line);
- lookup priority: ROM MD5 hash, then exact ROM basename without extension;
- the basename match is **case-sensitive**: keys are therefore stored both in
  their original case and lowercased (C# lookups are case-insensitive);
- arcade clones without their own data alias to their parent slug
  (`1943j -> 1943-the-battle-of-midway`), no empty `.MEM` is written.

---

## 8. Description (`desc`) Rules

- English, concise (max 96 chars), gameplay-oriented, no trailing period.
- Forbidden: double quotes, the `=` character, the word `address`
  (parser delimiters). The generator enforces this automatically.

---

## 9. Compliance Checklist

A `.MEM` file is compliant if:

- it returns the canonical `game`/`rom`/`events` Lua table with the
  canonical indentation;
- every entry uses a recognized type and condition (sections 2-3);
- `eq`/`neq` entries carry `value=`, `bit_true`/`bit_false` carry `mask=`;
- actions belong to the V11 vocabulary (or a composed prefix) and match
  `[A-Z0-9_]+`;
- families belong to the section 4 taxonomy;
- no entry is dead weight (`no_log`/`no_survey` true) unless intentionally
  generated with the `full` profile;
- `alias.json` targets an existing `.MEM` slug for every key.

`tools/mem-curator/validate_mem_repo.py` checks all of the above.
